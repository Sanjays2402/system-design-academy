Recommended path

Build the mental model in four moves.

Start with a compact read-heavy service, add distributed enforcement, learn ordered logs, then finish with a correctness-critical transaction system.

01 · Start with scale

URL Shortener

IDs, cache-aside reads, partitioning, and asynchronous analytics.

02 · Protect the edge

Rate Limiter

Distributed counters, failure policy, atomic updates, and tenant fairness.

03 · Reason about order

Message Queue

Partitions, durability, replication, consumer groups, and replay.

04 · Protect the invariant

Payment System

Idempotency, state machines, double-entry ledgers, and reconciliation.
